# Support

## How to Get Help

Thank you for using Orion Digital Solutions' projects and tools. Here's how to find the right level of support for your needs.

---

## Before Asking for Help

Please check the following resources first — your question may already be answered:

1. **Repository README** — Setup instructions, prerequisites, and quick-start guides
2. **Repository Wiki** *(if available)* — In-depth documentation for the specific project
3. **Open and closed Issues** — Search existing issues; someone may have reported the same problem
4. **CONTRIBUTING.md** — Guidance on development setup and common workflows

---

## Reporting a Bug

If you have found a bug, please **open an issue** in the relevant repository using the **Bug Report** template.

Provide as much detail as possible:
- Steps to reproduce
- Expected vs. actual behavior
- Environment details (OS, runtime version, dependency versions)
- Relevant logs or error messages

> **Security vulnerabilities** must not be reported as public issues. Please follow our [Security Policy](SECURITY.md) instead.

---

## Requesting a Feature

For feature requests, open an issue using the **Feature Request** template and describe:
- The problem you're trying to solve
- Your proposed solution
- Any alternatives you've considered

---

## General Questions & Discussions

For questions that are not bugs or feature requests:

- Use **GitHub Discussions** *(if enabled)* in the relevant repository for open-ended questions and community conversation
- Review existing discussions before opening a new one
